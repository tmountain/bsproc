Bayespam.rb and bsproc Bayesian spam filter utility
Designed with Paul Graham's A Plan for Spam in mind. A bayesian spam filter library bayespam.rb provides a process for indexing your current email and parsing each new email to discriminate your mail from unsolicited crap. Bsproc is a command line tool that will create an index and can be run by procmail on incoming mail. We got some lexical analysis hints from Eric S. Raymond's bogofilter.

Bayespam.rb and bsproc are both Ruby code and thus will require Ruby be installed wherever you intend to use either. We use bsproc on various flavors of Linux. We have not tested it on any other platforms. Let us know if you have. Bayespam.rb and bsproc are free software licensed under the GPL.

We are now working on remodeling after Paul Graham's Better Bayesian Filtering.
Updates
04.22.03 : 0.2.10 is released. After ignoring bsproc for a few months we have been reinvigorated by the potential of some new developers. This release cleans up the object API a bit and adds a lot of comments to the code. This was all taken care of by Travis to help people start hacking at bsproc. A new developer (I will make available his name and email when he OKs it) has shown a lot of effort in making bsproc work more practically. We are discussing changes like making the index evolutionary and making the index a BDB disk hash. The next release will have some major changes.
02.21.03 : 0.2.9 is released. In further proof of his indelible speed freakishness Travis found that Ruby's Marshalling that we were using for the cache was making up most of the execution time when generating the cache. So now it dumps straight to ASCII.

0.2.8 is released. Travis went insane trying to cram the indexing process of bsproc into an ever decreasing number of seconds. It started innocently enough with this post to ruby-talk and by the end of it, with a dose of help from several people on the list, we end up with a C optimized hash.succ! function (contributed by MoonWolf) that has to be compiled on installation. Make sure to read the bsproc-latest/README for installation instructions.
02.18.03 : 0.2.7 is released. Travis made some philosophical updates to limit bsproc's liberal consumption of memory. Base64 encoded messages are now decoded and their contents scrutinized. Bundles of efficiency enhancements.
